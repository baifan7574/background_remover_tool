# 🎁 邀请分享系统设计方案

## 📋 系统概述
设计一个多层次的邀请分享机制，通过奖励机制促进用户增长和活跃度。

## 🎯 邀请奖励机制

### 1. 基础邀请奖励
```
邀请成功注册：
├── 邀请人：获得 5次免费使用
├── 被邀请人：获得 3次免费使用
└── 有效期：永久有效
```

### 2. 激活奖励
```
被邀请人首次使用工具：
├── 邀请人：额外获得 3次免费使用
├── 被邀请人：额外获得 2次免费使用
└── 条件：注册后7天内激活
```

### 3. 消费奖励
```
被邀请人首次付费：
├── 邀请人：获得现金奖励 10%
├── 被邀请人：获得优惠券 ¥10
└── 条件：注册后30天内付费
```

### 4. 团队奖励
```
邀请人数达到里程碑：
├── 5人：额外获得 20次免费使用
├── 10人：升级为铜牌代理 (15%佣金)
├── 50人：升级为银牌代理 (20%佣金)
└── 100人：升级为金牌代理 (25%佣金)
```

## 📊 数据库设计

### 邀请记录表
```sql
CREATE TABLE invite_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    inviter_id INTEGER,                    -- 邀请人ID
    invitee_id INTEGER,                    -- 被邀请人ID
    invite_code VARCHAR(20) UNIQUE,        -- 邀请码
    invite_source VARCHAR(50),             -- 邀请来源 (微信/QQ/链接)
    invite_channel VARCHAR(50),            -- 邀请渠道
    status ENUM('pending', 'registered', 'activated', 'converted') DEFAULT 'pending',
    ip_address VARCHAR(45),                 -- 注册IP
    user_agent TEXT,                        -- 用户代理
    registered_at TIMESTAMP,                -- 注册时间
    activated_at TIMESTAMP,                 -- 激活时间
    converted_at TIMESTAMP,                 -- 付费时间
    reward_granted BOOLEAN DEFAULT FALSE,   -- 奖励是否发放
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (inviter_id) REFERENCES users(id),
    FOREIGN KEY (invitee_id) REFERENCES users(id)
);
```

### 奖励发放记录表
```sql
CREATE TABLE reward_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,                       -- 获奖用户ID
    invite_record_id INTEGER,              -- 关联邀请记录
    reward_type ENUM('register', 'activate', 'purchase', 'milestone'), -- 奖励类型
    reward_amount INTEGER,                  -- 奖励数量 (次数或金额)
    reward_description TEXT,                -- 奖励描述
    status ENUM('pending', 'granted', 'expired') DEFAULT 'pending',
    granted_at TIMESTAMP,                   -- 发放时间
    expires_at TIMESTAMP,                   -- 过期时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (invite_record_id) REFERENCES invite_records(id)
);
```

### 邀请等级表
```sql
CREATE TABLE invite_levels (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    level_name VARCHAR(20),                -- 等级名称
    min_invites INTEGER,                    -- 最小邀请人数
    max_invites INTEGER,                    -- 最大邀请人数
    commission_rate DECIMAL(5,2),           -- 佣金比例
    monthly_bonus INTEGER,                  -- 月度奖励次数
    benefits JSON,                          -- 其他权益
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 🔧 技术实现方案

### 1. 邀请码生成
```python
import random
import string

def generate_invite_code(user_id):
    """生成唯一邀请码"""
    prefix = "CBT"  # Cross-Border Tools
    suffix = ''.join(random.choices(string.ascii_uppercase + string.digits, k=6))
    return f"{prefix}{suffix}"

def is_invite_code_available(code):
    """检查邀请码是否可用"""
    return not InviteRecord.query.filter_by(invite_code=code).first()
```

### 2. 邀请链接生成
```python
def generate_invite_link(user_id, channel='default'):
    """生成邀请链接"""
    invite_code = generate_invite_code(user_id)
    base_url = "https://your-domain.com"
    return f"{base_url}/register?invite={invite_code}&channel={channel}"
```

### 3. 奖励发放逻辑
```python
def grant_invite_reward(invite_record, reward_type):
    """发放邀请奖励"""
    inviter = invite_record.inviter
    
    if reward_type == 'register':
        # 注册奖励
        grant_usage_bonus(inviter.id, 5, "邀请注册奖励")
        grant_usage_bonus(invite_record.invitee_id, 3, "被邀请注册奖励")
        
    elif reward_type == 'activate':
        # 激活奖励
        grant_usage_bonus(inviter.id, 3, "邀请激活奖励")
        grant_usage_bonus(invite_record.invitee_id, 2, "被邀请激活奖励")
        
    elif reward_type == 'purchase':
        # 付费奖励
        grant_cash_reward(inviter.id, 10, "邀请付费奖励")
        grant_coupon(invite_record.invitee_id, 10, "被邀请付费优惠券")
```

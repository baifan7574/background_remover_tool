## 🔍 验证数据真实性

### 方法1：查看浏览器控制台

**打开浏览器开发者工具**（按F12），查看控制台输出：

**如果使用真实数据**：
- 不应该看到错误信息
- 应该看到正常的API请求

**如果使用模拟数据**：
- 可能会看到警告信息
- 功能仍然可用，但数据是模拟的

### 方法2：查看后端服务器日志

**查看后端服务器终端窗口**，当调用关键词分析功能时：

**如果使用真实数据**：
```
🔍 使用Groq API提取关键词...
✅ Groq API关键词提取成功
```
或
```
🔍 使用Google Trends查询竞争度: wireless bluetooth headphones
✅ Google Trends竞争度查询成功
```

**如果使用模拟数据**：
```
⚠️ Groq API调用失败，使用模拟数据
```
或
```
⚠️ Google Trends未找到数据，使用模拟数据
```

### 方法3：查看结果中的`source`字段

**在结果中查找`source`字段**：

- `source: "groq"` = 使用了Groq AI（真实AI）✅
- `source: "google_trends"` = 使用了Google Trends（真实数据）✅
- `source: "mock"` = 使用了模拟数据 ⚠️
- `source: "mixed"` = 部分真实，部分模拟 ⚠️

## ❓ 常见问题

### Q1：为什么终端显示库未安装？

**原因**：
- 可能安装到了不同的Python环境
- 可能需要重启终端

**解决**：
1. 确认使用正确的Python环境：
   ```bash
   python --version
   where python
   ```
2. 重新安装依赖：
   ```bash
   pip install --upgrade groq pytrends
   ```
3. 验证安装：
   ```bash
   python -c "import groq; import pytrends; print('OK')"
   ```

### Q2：为什么前端还是报错？

**原因**：
- 浏览器缓存问题
- 前端代码未更新

**解决**：
1. **强制刷新浏览器**：按 `Ctrl+F5`
2. **清除浏览器缓存**：
   - Chrome: 按 `Ctrl+Shift+Delete`，清除缓存
   - 或者：打开开发者工具，右键刷新按钮，选择"清空缓存并硬性重新加载"

### Q3：为什么功能仍然显示"模拟数据"？

**原因**：
- 依赖未正确安装
- 后端服务器未重启
- API调用失败

**解决**：
1. 检查依赖是否安装：
   ```bash
   pip list | findstr "groq pytrends"
   ```
2. 重启后端服务器
3. 查看后端日志，确认是否使用了真实API

# 🚀 服务器启动文件说明

## 📋 启动文件清单

### 1. **后端服务器启动文件**

**文件位置**: `sk_app.py` (项目根目录)

**启动方式**:
```bash
python sk_app.py
```

**或者使用批处理文件**:
```bash
启动后端服务器.bat
```

**端口**: `5000`
**访问地址**: `http://localhost:5000`

---

### 2. **前端服务器启动文件**

**前端文件位置**: `frontend/index.html` (主页面)

**启动方式**:
```bash
cd frontend
python -m http.server 8000
```

**或者使用批处理文件**:
```bash
启动前端服务器.bat
```

**端口**: `8000`
**访问地址**: `http://localhost:8000`

---

## ✅ 启动检查清单

### 启动后端前检查：
- [ ] Python已安装（Python 3.7+）
- [ ] 已安装依赖：`pip install -r requirements.txt`
- [ ] 端口5000未被占用

### 启动前端前检查：
- [ ] 后端服务器已启动
- [ ] `frontend` 文件夹存在
- [ ] 端口8000未被占用

## 🐛 常见问题

### 问题1：端口被占用

**错误信息**:
```
Address already in use
端口5000已被占用
```

**解决方法**:
1. 找到占用端口的进程并关闭
2. 或者修改端口号（需要同时修改前端和后端的配置）

**Windows查找占用端口**:
```bash
netstat -ano | findstr :5000
```

---

### 问题2：模块未找到

**错误信息**:
```
ModuleNotFoundError: No module named 'xxx'
```

**解决方法**:
```bash
pip install -r requirements.txt
```

---

### 问题3：前端无法连接后端

**检查**:
1. 后端是否正常运行
2. 后端地址是否正确（默认：`http://localhost:5000`）
3. 检查浏览器控制台错误信息

---

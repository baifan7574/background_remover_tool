# 📊 数据库连接和部署说明

## 问题1：数据库连接情况

### 当前使用的数据库类型

你的系统支持**两种数据库**：

#### 1. **本地JSON文件数据库**（当前使用）

**位置**：`data/` 目录下的JSON文件

**文件**：
- `data/users.json` - 用户账号数据
- `data/profiles.json` - 用户资料数据
- `data/usage.json` - 工具使用记录

**如何确认**：
- 后端启动时看到：`✅ 数据持久化管理器已加载`
- 检查 `data/` 目录，应该有JSON文件

**数据保存**：
- ✅ **是的，数据会保存**
- 用户注册、登录信息都会保存在 `users.json`
- 使用记录会保存在 `usage.json`
- 每次操作后会自动保存

#### 2. **Supabase云端数据库**（可选）

**位置**：云端服务器

**如何启用**：
- 需要配置 `.env` 文件
- 设置 `SUPABASE_URL` 和 `SUPABASE_KEY`
- 后端会优先尝试连接Supabase

**当前状态**：
- 如果没有配置Supabase，会自动使用本地JSON文件
- 本地测试通常使用JSON文件更方便

## 📋 完整部署步骤

### 第1步：准备服务器环境

#### 选择服务器平台：

**推荐**：
- **PythonAnywhere**（简单，适合Python应用）
- **Heroku**（免费套餐已取消，需要付费）
- **VPS**（如阿里云、腾讯云，需要自己配置）

#### 服务器要求：

- Python 3.8+
- 磁盘空间：至少1GB
- 内存：至少512MB
- 支持HTTPS（如果有域名）

---

### 第2步：上传文件到服务器

#### 需要上传的文件：

**必需文件**：
```
background_remover_tool/
├── sk_app.py              # 后端主程序
├── data_manager.py        # 数据管理器
├── supabase_db.py         # Supabase连接（可选）
├── requirements.txt       # Python依赖列表（需要创建）
├── .env                   # 环境变量配置（如果有）
└── frontend/              # 前端文件
    ├── index.html
    ├── css/
    ├── js/
    └── ...
```

**不需要上传**：
- `data/` 目录（本地测试数据，上传后会在服务器上重新创建）
- `*.bat` 文件（Windows批处理文件，Linux不需要）
- `*.md` 文档文件（可选，不影响运行）

---

### 第3步：在服务器上安装依赖

#### 创建 `requirements.txt`：

```txt
Flask==2.3.3

opencv-python==4.8.1.78
python-dotenv==1.0.0
supabase==2.0.0
werkzeug==2.3.7
```

#### 安装依赖：

```bash
pip install -r requirements.txt
```

---

### 第4步：配置环境变量

#### 创建 `.env` 文件（如果需要Supabase）：

```env
# Supabase配置（可选）
SUPABASE_URL=你的supabase_url
SUPABASE_KEY=你的supabase_key

# 应用配置
FLASK_ENV=production
SECRET_KEY=你的密钥
```

#### 如果只用本地JSON数据库：

- 不需要配置Supabase
- 系统会自动使用 `DataManager`（本地JSON文件）

---

###  第5步：修改代码配置

#### 需要修改的地方：

**1. `sk_app.py` - CORS配置**

需要添加你的域名：

```python
CORS(app, 
     resources={r"/api/*": {
         "origins": [
             "http://localhost:8000", 
             "http://127.0.0.1:8000",
             "https://你的域名.com",  # 添加你的域名
             "http://你的域名.com"    # 添加HTTP版本（如果有）
         ],
         # ...
     }}
)
```

**2. `frontend/js/app.js` - API地址**

修改 `getApiBaseUrl` 函数：

```javascript
function getApiBaseUrl() {
    // 如果是生产环境，使用你的域名
    if (window.location.hostname !== 'localhost' && 
        window.location.hostname !== '127.0.0.1') {
        return 'https://你的域名.com';  // 或 http://你的域名.com
    }
    return 'http://localhost:5000';  // 本地开发
}
```

---

### 第6步：配置服务器运行

#### PythonAnywhere：

1. **上传文件**（通过Web界面或Git）
2. **创建Web App**：
   - 选择 "Manual configuration"
   - Python版本：3.10
3. **配置WSGI文件**：
   ```python
   import sys
   path = '/home/你的用户名/background_remover_tool'
   if path not in sys.path:
       sys.path.append(path)
   
   from sk_app import app as application
   ```
4. **设置静态文件**：
   - URL: `/static/`
   - 目录: `/home/你的用户名/background_remover_tool/frontend`
5. **安装依赖**：在Bash控制台运行 `pip install -r requirements.txt`
6. **重启Web App**

#### VPS（使用systemd）：

创建服务文件 `/etc/systemd/system/background-tool.service`：

```ini
[Unit]
Description=Background Remover Tool
After=network.target

[Service]
User=你的用户名
WorkingDirectory=/path/to/background_remover_tool
Environment="PATH=/usr/bin:/usr/local/bin"
ExecStart=/usr/bin/python3 sk_app.py
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl start background-tool
sudo systemctl enable background-tool
```

### 第7步：配置域名和HTTPS

#### 1. **购买域名**（可选）

- 如果没有域名，可以使用服务器提供的默认地址

#### 2. **配置DNS**

- 将域名A记录指向服务器IP地址

#### 3. **配置HTTPS**（推荐）

**使用Let's Encrypt（免费）**：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名.com
```

**或使用Nginx反向代理**：

```nginx
server {
    listen 80;
    server_name 你的域名.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 📊 数据迁移（如果需要）

### 从本地JSON迁移到Supabase：

1. **配置Supabase**：
   - 创建Supabase项目
   - 配置 `.env` 文件
   - 创建数据库表

2. **导出本地数据**：
   ```python
   # 读取 data/users.json
   import json
   with open('data/users.json', 'r') as f:
       users = json.load(f)
   ```

3. **导入到Supabase**：
   ```python
   # 使用Supabase客户端导入数据
   for user_id, user_data in users.items():
       supabase.table('users').insert(user_data).execute()
   ```

## 📋 在码支付平台需要配置的内容

### **重要：您需要做的配置（2项）**

#### 1. **配置异步通知地址（必须）**

**什么是异步通知地址？**
- 用户支付成功后，码支付平台会自动POST请求到这个地址
- 您的服务器接收通知后，自动更新订单状态、激活会员

**如何配置：**

1. 登录码支付平台（`pay.mzfpay.com`）
2. 进入 **"商户中心"** 或 **"账号设置"**
3. 找到 **"异步通知地址"** 或 **"Notify URL"** 设置项
4. 填写您的服务器地址：

   **开发测试环境（本地）：**
   ```
   无法配置（本地地址无法接收回调）
   ```
   
   **正式环境（上线后）：**
   ```
   https://yourdomain.com/api/payment/notify
   ```
   ⚠️ 必须是 HTTPS 地址（不能是 HTTP）
   ⚠️ 必须是公网可访问的地址（不能是 localhost）

5. 保存配置

**示例：**
- ✅ 正确：`https://www.yourdomain.com/api/payment/notify`
- ✅ 正确：`https://api.yourdomain.com/payment/notify`
- ❌ 错误：`http://localhost:5000/api/payment/notify`（本地地址）
- ❌ 错误：`http://yourdomain.com/api/payment/notify`（必须HTTPS）

---

#### 2. **配置同步返回地址（可选但推荐）**

**什么是同步返回地址？**
- 用户支付完成后，浏览器自动跳转回您的网站
- 用于显示支付成功页面，提升用户体验

**如何配置：**

1. 在码支付平台找到 **"同步返回地址"** 或 **"Return URL"** 设置项
2. 填写您的网站地址：

   **正式环境：**
   ```
   https://yourdomain.com/payment/success.html
   ```

**注意：**
- 同步返回地址不是必须的（我们会在创建订单时单独指定）
- 但配置全局默认地址更方便

## 🔧 本地开发测试配置

### 问题：本地无法接收回调通知

**本地开发时的问题：**
- 您的本地服务器地址是 `http://localhost:5000`
- 码支付平台无法访问您的本地地址
- 异步通知无法送达

### 解决方案

#### **方案1：使用内网穿透工具（推荐测试）**

使用工具将本地地址暴露到公网：

1. **使用 ngrok（免费）：**
   ```bash
   # 安装 ngrok
   # 下载：https://ngrok.com/download
   
   # 启动 ngrok
   ngrok http 5000
   
   # 会生成一个公网地址，例如：
   # https://abc123.ngrok.io -> http://localhost:5000
   ```

2. **在码支付平台配置：**
   ```
   异步通知地址：https://abc123.ngrok.io/api/payment/notify
   ```

3. **测试完成后：**
   - 使用真实域名替换 ngrok 地址

#### **方案2：手动验证支付（测试用）**

1. 创建订单后，获取支付链接
2. 手动完成支付
3. 手动调用激活会员接口（跳过回调）

**不推荐用于正式环境**

## 💡 常见问题

### **Q1: 本地开发无法接收回调怎么办？**

**A:** 
- 使用 ngrok 等内网穿透工具
- 或先实现代码，上线后再测试回调

### **Q2: 必须配置异步通知地址吗？**

**A:** 
- **必须配置**，否则无法自动更新订单状态
- 没有回调，订单状态会一直是"待支付"

### **Q3: 同步返回地址必须配置吗？**

**A:** 
- 不是必须的
- 我们可以在创建订单时单独指定
- 但配置全局默认地址更方便

### **Q4: 测试环境可以配置 localhost 吗？**

**A:** 
- 不可以
- 码支付平台无法访问 localhost
- 必须使用公网地址或 ngrok

# 🔧 支付签名错误修复说明

## ❌ 问题描述

**错误信息：**
```
📥 码支付响应状态码: 200
   响应内容: {"code":0,"msg":"签名错误","trade_no":""}
```

**签名原始字符串：**
```
money=299.00&name=企业版-月付&notify_url=http://localhost:5000/api/payment/notify&out_trade_no=ORDER20251124184624E91AF07C&pid=10294&return_url=http://localhost:8000/payment/success.html?out_trade_no=ORDER20251124184624E91AF07C&order_no=ORDER20251124184624E91AF07C&type=wxpay&key=X0cJyf2G0EjDKtQe9NMf
```

---

## 🔍 问题分析

### 可能的原因：

1. **return_url 中包含查询参数**
   - `return_url` 的值包含 `?out_trade_no=...&order_no=...`
   - 这可能导致签名计算时出现问题

2. **参数编码问题**
   - 某些易支付平台要求参数值进行URL编码
   - 或者不进行URL编码，但格式必须正确

3. **签名算法差异**
   - 不同的易支付平台可能有不同的签名规则
   - 需要确认码支付的具体要求

---

## ✅ 修复方案

### 方案1：简化 return_url（推荐）

移除 `return_url` 中的查询参数，在订单号中处理。

### 方案2：URL编码参数值

对 `return_url` 和 `notify_url` 进行URL编码。

### 方案3：检查参数顺序

确保参数按照ASCII码正确排序。

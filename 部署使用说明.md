# 🚀 部署和使用说明

## 📖 重要概念：开发环境和生产环境

### 🔵 开发环境（本地开发）
- **特点**：在你自己电脑上运行
- **用途**：开发和测试
- **需要**：本地启动后端和前端服务器
- **访问**：`http://localhost:8000`

### 🟢 生产环境（线上服务器）
- **特点**：已经部署到云服务器（如PythonAnywhere）
- **用途**：给真实用户使用
- **不需要**：本地不需要启动服务器
- **访问**：通过域名访问，如 `https://your-domain.com`

---

## ❓ 您的情况判断

### 情况1：后端已部署到线上服务器 ✅

如果您已经：
- ✅ 将后端代码部署到PythonAnywhere或其他云服务器
- ✅ 配置了域名（如 `https://baifan7574.pythonanywhere.com`）
- ✅ 服务器24小时运行

**那么：**
- ❌ **不需要**本地启动后端服务器
- ✅ **只需要**修改前端配置，指向线上API
- ✅ 用户可以直接访问线上网站使用

**如何修改：**
1. 修改 `frontend/js/auth.js` 第4行：
   ```javascript
   this.apiBaseUrl = 'https://your-domain.com'; // 改为您的线上地址
   ```

2. 修改 `frontend/js/app.js` 第6行：
   ```javascript
   this.apiBaseUrl = 'https://your-domain.com'; // 改为您的线上地址
   ```

3. 将前端文件上传到服务器或静态网站托管

---

### 情况2：只连接了数据库，后端还在本地 ❌

如果您：
- ✅ 连接到Supabase云数据库
- ❌ 但后端代码还在本地（`sk_app.py`）
- ❌ 还没有部署到服务器

**那么：**
- ✅ **需要**本地启动后端服务器
- ✅ **需要**本地启动前端服务器
- ✅ 用户只能在你电脑运行的时候访问

---

## 🎯 快速判断方法

### 检查步骤：

1. **检查后端是否已部署**
   - 问自己：后端代码 `sk_app.py` 是否已经上传到云服务器？
   - 如果没有 → 需要在本地启动

2. **检查是否有线上域名**
   - 问自己：是否有像 `https://xxx.pythonanywhere.com` 这样的地址？
   - 如果没有 → 需要在本地启动

3. **检查前端配置**
   - 打开 `frontend/js/auth.js`
   - 看第4行：`this.apiBaseUrl = 'http://localhost:5000'`
   - 如果还是 `localhost` → 说明还在本地开发模式

---

## 🔧 如果后端已部署，如何切换到线上模式

### 步骤1：修改前端配置

创建 `frontend/js/config.js` 文件（已创建）：
```javascript
// 自动检测环境
const isDevelopment = window.location.hostname === 'localhost';
const API_BASE_URL = isDevelopment 
    ? 'http://localhost:5000'  // 开发环境
    : 'https://your-domain.com'; // 生产环境（改为您的实际地址）
```

### 步骤2：修改前端文件使用配置

修改 `frontend/js/auth.js`：
```javascript
// 在文件开头添加
const API_BASE_URL = window.location.hostname === 'localhost' 
    ? 'http://localhost:5000' 
    : 'https://your-domain.com'; // 改为您的实际地址

class AuthManager {
    constructor() {
        this.apiBaseUrl = API_BASE_URL; // 使用配置
        // ...
    }
}
```

修改 `frontend/js/app.js`：
```javascript
class AppManager {
    constructor() {
        // ...
        this.apiBaseUrl = API_BASE_URL; // 使用配置
        // ...
    }
}
```

### 步骤3：部署前端

- **方式1**：上传到与后端相同的服务器
- **方式2**：使用静态网站托管（GitHub Pages、Vercel、Netlify等）

## 💡 推荐方案

### 如果还没有部署：

1. **继续本地开发**：使用 `start_dev_servers.bat` 启动
2. **测试功能**：确保所有功能正常
3. **准备部署**：完成后端部署到云服务器

### 如果已经部署：

1. **修改前端配置**：指向线上API地址
2. **部署前端**：上传到服务器或静态托管
3. **无需本地启动**：直接使用线上版本

# 🔧 支付订单创建500错误调试指南

## 问题描述

创建支付订单时返回500内部服务器错误。

## 🔍 调试步骤

### 步骤1：查看后端日志

**最重要！** 查看运行 `python sk_app.py` 的终端窗口，应该能看到详细的错误信息：

**正常情况应该看到：**
```
🔄 准备调用码支付接口...
   订单号: ORDER...
   金额: ¥19
   商品名称: 入门版-月付
   支付方式: alipay
   通知地址: http://localhost:5000/api/payment/notify
   返回地址: http://localhost:8000/payment/success.html?...
📤 调用码支付API: https://pay.mzfpay.com/xpay/epay/mapi.php
   参数: {...}
📥 码支付响应状态码: 200
   响应内容: {...}
```

**如果看到错误：**
- ❌ `码支付客户端初始化失败` → 检查导入问题
- ❌ `调用码支付接口异常` → 查看具体错误信息
- ❌ `码支付响应状态码: 非200` → 检查API地址和参数
- ❌ `响应不是JSON格式` → 码支付返回了错误页面

---

### 步骤2：检查码支付客户端初始化

**查看后端启动时的日志：**

应该看到：
```
✅ 码支付客户端已初始化
   商户ID: 10294
```

如果没有看到，说明初始化失败，检查：
- `backend/mzfpay_client.py` 文件是否存在
- 是否有导入错误

---

### 步骤3：常见错误原因

#### 错误1：码支付API返回非JSON格式

**症状：**
- 后端日志显示：`⚠️ 响应不是JSON格式`
- 可能是HTML错误页面

**可能原因：**
- API地址错误
- 参数格式错误
- 签名错误

**解决方法：**
- 检查API地址是否正确
- 检查签名算法是否正确
- 查看码支付返回的原始内容

---

#### 错误2：网络请求失败

**症状：**
- 后端日志显示：`网络请求失败: ...`
- 可能是超时或连接错误

**可能原因：**
- 网络连接问题
- 码支付服务器不可访问
- 防火墙阻止

**解决方法：**
- 检查网络连接
- 尝试直接访问码支付API地址
- 检查防火墙设置

---

#### 错误3：签名验证失败

**症状：**
- 码支付返回：`签名错误` 或 `sign error`

**可能原因：**
- 签名算法不正确
- 参数排序错误
- 商户密钥错误

**解决方法：**
- 检查签名生成逻辑
- 确认商户密钥正确
- 参考码支付文档验证签名算法

---

#### 错误4：参数错误

**症状：**
- 码支付返回：`参数错误` 或 `参数不完整`

**可能原因：**
- 缺少必填参数
- 参数格式不正确
- 参数值无效

**解决方法：**
- 检查所有必填参数是否都传递了
- 确认参数格式符合码支付要求
- 查看码支付API文档

---

## 🛠️ 临时解决方案

如果暂时无法解决，可以**临时禁用码支付**，使用模拟数据：

### 方法1：修改初始化代码

在 `sk_app.py` 中，临时禁用码支付：

```python
# 临时禁用码支付（用于调试）
MZFPAY_AVAILABLE = False
mzfpay_client = None
```

这样会返回模拟订单数据，不会调用真实API。

---

### 方法2：添加错误处理降级

代码已经实现了降级处理：如果码支付调用失败，会返回模拟数据。

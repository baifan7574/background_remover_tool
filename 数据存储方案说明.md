# 💾 数据存储方案说明

**更新时间**: 2025-11-25

---

## 📊 当前数据存储方式

### ✅ JSON 文件存储（当前使用）

**存储位置**: 服务器上的 `data/` 目录

**文件列表**:
- `data/users.json` - 用户数据
- `data/profiles.json` - 用户资料
- `data/usage.json` - 使用记录
- `data/invites.json` - 邀请关系
- `data/orders.json` - 订单数据

**特点**:
- ✅ **数据是持久化的**，不会消失
- ✅ 服务器重启后数据仍然存在
- ✅ 简单易用，不需要数据库配置
- ✅ 适合中小型应用
- ⚠️ 不适合超大规模并发（但您的应用目前完全够用）

---

## ❓ 数据会消失吗？

### 答案：**不会消失**

**原因**:
1. **数据保存在文件中**
   - 所有数据都保存在服务器硬盘上的 JSON 文件中
   - 这些文件是永久存储的

2. **自动保存机制**
   - 每次数据更新时，`data_manager.py` 会自动保存到文件
   - 包括：用户注册、会员激活、订单创建、使用记录等

3. **服务器重启不影响**
   - 服务器重启后，文件仍然存在
   - 应用启动时会自动加载这些文件

---

## 🔄 两种存储方案对比

### 方案1: JSON 文件存储（当前）

**优点**:
- ✅ 简单，不需要数据库配置
- ✅ 数据直接可见（可以打开 JSON 文件查看）
- ✅ 备份方便（直接复制文件）
- ✅ 适合中小型应用（用户数 < 10万）

**缺点**:
- ⚠️ 不适合超大规模并发（但您目前完全够用）
- ⚠️ 没有事务支持（但对于您的应用影响不大）

**适用场景**: 
- ✅ 个人/小团队项目
- ✅ 用户数 < 10万
- ✅ 不需要复杂查询

---

### 方案2: 数据库存储（Supabase）

**优点**:
- ✅ 支持大规模并发
- ✅ 有事务支持
- ✅ 支持复杂查询
- ✅ 数据更安全（自动备份）

**缺点**:
- ⚠️ 需要配置数据库连接
- ⚠️ 需要学习 SQL（可选）
- ⚠️ 稍微复杂一些

**适用场景**:
- ✅ 大型项目
- ✅ 用户数 > 10万
- ✅ 需要复杂查询

---

## 💡 建议

### 当前阶段：继续使用 JSON 文件存储

**理由**:
1. ✅ 您的应用刚起步，用户数不会太多
2. ✅ JSON 文件存储完全够用
3. ✅ 简单可靠，不需要额外配置

**什么时候需要迁移**:

- 需要复杂的数据查询
- 需要数据分析和报表
- 需要多服务器部署

## 📋 数据备份建议

### 即使使用 JSON 文件，也要定期备份

**备份方法**:

1. **手动备份**
   ```bash
   # 在 PythonAnywhere Console 中执行
   cd ~/background_remover_tool
   tar -czf data_backup_$(date +%Y%m%d).tar.gz data/
   ```

2. **下载备份**
   - 在 Files 页面下载 `data/` 目录
   - 保存到本地

3. **定期备份**
   - 建议每周备份一次
   - 或使用 PythonAnywhere 的自动备份功能

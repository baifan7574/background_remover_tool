# 📖 完整使用说明 - 从零开始

## 🎯 简单理解

### 前端 vs 后端 - 简单比喻

**前端（端口8000）** = 你看到的网页界面
- 🖥️ 用户看到的页面（HTML、CSS、JavaScript）
- 👆 点击按钮、上传图片、查看结果
- 📱 **这就是用户访问的地址：http://localhost:8000**

**后端（端口5000）** = 后台处理程序
- ⚙️ 处理业务逻辑（Python Flask）
- 🖼️ 处理图片、调用AI、连接数据库
- 🔧 **这是API服务器，不需要直接访问**

## 🚀 启动流程

### 第1步：启动服务器

运行启动脚本：
```bash

```

或者分别启动：

**终端1 - 启动后端（必须）**


## 🔌 端口说明

### 端口8000（前端）- 用户访问这个！

**地址**: `http://localhost:8000`

**作用**:
- ✅ 显示网页界面
- ✅ 用户点击、上传、查看结果
- ✅ 发送请求到后端API

**你作为用户**：
- ✅ **必须访问这个端口**
- ✅ 在这里注册、登录、使用工具
- ✅ 这是你看到的网页

**不要直接访问**: 后端端口5000（那是API服务器）

---

### 端口5000（后端）- 后台自动工作

**地址**: `http://localhost:5000`（通常不需要直接访问）

**作用**:
- ⚙️ 接收前端请求
- 🖼️ 处理图片（背景移除、压缩等）
- 💾 保存数据到数据库
- 🔐 验证用户登录

**你作为用户**：
- ❌ **不需要直接访问这个端口**
- ✅ 前端会自动调用这个API
- ✅ 你只需要访问8000端口

**什么时候需要看5000端口？**
- 🔍 查看后端日志（看处理进度）
- 🐛 调试错误（看后端错误信息）

## 📋 完整的测试流程

### 第1步：确保服务器都在运行

**检查后端（端口5000）**：
- 打开运行 `python sk_app.py` 的命令窗口
- 应该看到：`Running on http://127.0.0.1:5000`
- 如果没运行，启动它

**检查前端（端口8000）**：
- 打开运行 `python -m http.server 8000` 的命令窗口
- 应该看到：`Serving HTTP on :: port 8000`
- 如果没运行，启动它

### 第2步：在浏览器打开前端

1. **打开浏览器**（Chrome、Edge、Firefox等）

2. **访问前端地址**：
   ```
   http://localhost:8000
   ```
   ⚠️ **记住：作为用户，你只访问8000端口！**

3. **应该看到网页界面**：
   - 顶部导航栏
   - "PixelMagic 跨境电商处理工具"标题
   - 工具卡片（背景移除、图片压缩等）

### 第3步：注册/登录

**注册新账号**：
1. 点击右上角"注册"按钮
2. 填写完整信息：
   - 姓名：例如"张三"
   - 邮箱：例如"test@example.com"
   - 密码：至少6位
   - 确认密码：和密码一致
3. 点击"注册"
4. 应该看到"注册成功！欢迎使用！"

**如果提示"用户已存在"**：
- 说明这个邮箱已注册
- 直接点击"登录"按钮
- 输入邮箱和密码登录

### 第4步：使用背景移除工具

1. **点击"背景移除"工具卡片**（有大面具图标的那个）

2. **上传图片**：
   - 点击上传区域
   - 或拖拽图片到上传区域
   - 选择一张有清晰主体的图片（人物、产品等）

3. **点击"开始处理"**（或选择图片后自动开始）

4. **等待处理**：
   - 前端显示进度条（0% → 100%）
   - 后端控制台显示：`✅ 使用rembg库进行背景移除...`
   - 首次使用会下载AI模型（需要时间）

5. **查看结果**：
   - 应该看到处理后的图片（背景被移除）
   - 应该显示"下载图片"按钮
   - 应该显示使用统计（今日使用/剩余次数）

6. **下载图片**：
   - 点击"下载图片"按钮
   - 图片保存到下载文件夹

## ❓ 常见问题

### Q1: 我应该访问哪个端口？

**A**: **作为用户，你只访问8000端口！**
- ✅ 访问：`http://localhost:8000`
- ❌ 不要访问：`http://localhost:5000`（那是API服务器）

### Q2: 为什么需要两个服务器？

**A**: 
- **前端（8000）** = 用户看到的界面
- **后端（5000）** = 处理业务逻辑和图片

它们分工合作：
- 用户在8000端口操作界面
- 前端发送请求到5000端口的API
- 后端处理完成后返回结果给前端
- 前端显示结果给用户

### Q3: 如果关闭了服务器窗口怎么办？

**A**: 
- **关闭后端** = 无法处理请求，需要重启
- **关闭前端** = 无法访问网页，需要重启
- 两个都必须运行！

### Q4: 如何确认后端正在工作？

**A**: 
- 查看后端控制台窗口
- 应该看到：`Running on http://127.0.0.1:5000`
- 处理图片时应该看到处理日志

### Q5: 如何处理图片时没有反应？

**A**: 检查：
1. 后端服务器是否运行？
2. 后端控制台是否有错误？
3. 浏览器控制台（F12）是否有错误？
4. 是否已登录？

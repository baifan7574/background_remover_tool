# 💰 第三方支付平台接入原理与实现方案

## 📋 一、支付接入原理概述

### 1. **什么是第三方支付平台？**

第三方支付平台（如"码支付"）是一个**聚合支付服务商**，它：
- 整合了多个支付渠道（支付宝、微信支付、QQ钱包等）
- 为商户提供统一的支付接口
- 处理支付请求、回调通知、订单查询等

**优势**：
- ✅ 无需单独申请支付宝/微信商户账号
- ✅ 统一接口，接入简单
- ✅ 支持多种支付方式
- ✅ 自动处理支付回调

---

### 2. **支付流程原理**

```
用户下单 → 商户系统 → 第三方支付平台 → 支付渠道（支付宝/微信） → 用户支付
                ↑                                                          ↓
                └────────────────── 支付回调通知 ←─────────────────────────┘
```

**详细流程**：

1. **用户下单**
   - 用户在您的网站选择会员计划
   - 点击"立即支付"

2. **商户系统创建订单**
   - 后端生成订单号
   - 调用第三方支付平台API创建支付订单
   - 获取支付链接或二维码

3. **跳转到支付页面**
   - 用户跳转到第三方支付平台页面
   - 选择支付方式（支付宝/微信）
   - 完成支付

4. **支付回调通知**
   - 支付成功后，第三方平台会通知您的服务器
   - 有两种通知方式：
     - **同步通知**：用户支付完成后立即跳转回您的网站（可能不可靠）
     - **异步通知**：第三方平台后台主动通知您的服务器（可靠）

5. **订单状态更新**
   - 收到回调后，验证签名
   - 更新订单状态为"已支付"
   - 激活会员权益

## 🔧 二、码支付接口说明

### 1. **易支付接口（推荐）**

**特点**：
- ✅ 支持同步/异步通知
- ✅ 支持PC/移动端支付
- ✅ 提供完整支付状态
- ✅ 支持表单提交和API调用两种方式

**接口地址**：
- **Submit提交**：`https://pay.mzfpay.com/xpay/epay/submit.php`

  - 适合PC端支付
  
- **MApi提交**：`https://pay.mzfpay.com/xpay/epay/mapi.php`

**接口地址**：
- **标准支付接口**：`https://pay.mzfpay.com/xpay/hupijiao`
- **直接支付接口**：`https://pay.mzfpay.com/xpay/hupijiao/payment/do.html`

### 步骤2：理解支付参数

**创建支付订单需要的参数**：

| 参数名 | 说明 | 示例 |
|--------|------|------|
| `pid` | 商户ID | `10294` |
| `key` | 商户密钥 | `X0cJ...9NMf` |
| `type` | 支付方式 | `alipay`（支付宝）、`wxpay`（微信） |
| `out_trade_no` | 商户订单号 | `ORDER20251117123456` |
| `notify_url` | 异步通知地址 | `https://yourdomain.com/api/payment/notify` |
| `return_url` | 同步返回地址 | `https://yourdomain.com/payment/success` |
| `name` | 商品名称 | `AI背景移除工具-入门版` |
| `money` | 支付金额 | `19.00`（单位：元） |
| `sign` | 签名 | 根据参数生成的MD5签名 |
| `sign_type` | 签名类型 | `MD5` |

**签名生成规则**：
```
sign = MD5(pid + type + out_trade_no + notify_url + name + money + key)
```

---

### 步骤3：同步通知 vs 异步通知

#### **同步通知（return_url）**

**特点**：
- 用户支付完成后，浏览器自动跳转到您的网站
- 通过URL参数传递支付结果
- ⚠️ **不可靠**：用户可能关闭页面，网络可能中断

**用途**：
- 显示支付成功页面
- 提升用户体验

#### **异步通知（notify_url）**

**特点**：
- 第三方平台后台主动POST请求到您的服务器
- 即使页面关闭也会通知
- ✅ **可靠**：必须使用异步通知更新订单状态

**用途**：
- 更新订单状态
- 激活会员权益
- 发送通知邮件

**重要**：
- 异步通知必须返回 `success` 字符串，否则会重复通知
- 必须验证签名，防止伪造通知

## 💻 四、代码实现方案

### 1. **创建支付订单（后端）**

```python
import hashlib
import requests
from datetime import datetime

class MzfPayClient:
    """码支付客户端"""
    
    def __init__(self, merchant_id, merchant_key):
        self.merchant_id = merchant_id  # 商户ID: 10294
        self.merchant_key = merchant_key  # 商户密钥
        self.api_url = "https://pay.mzfpay.com/xpay/epay/mapi.php"  # API接口
        self.submit_url = "https://pay.mzfpay.com/xpay/epay/submit.php"  # 表单提交接口
    
    def generate_sign(self, params):
        """生成签名"""
        # 按照参数名排序
        sorted_params = sorted(params.items())
        # 拼接字符串（排除sign和sign_type）
        sign_str = ''.join([f"{k}={v}" for k, v in sorted_params if k not in ['sign', 'sign_type']])
        # 加上商户密钥
        sign_str += self.merchant_key
        # MD5加密
        return hashlib.md5(sign_str.encode('utf-8')).hexdigest()
    
    def create_payment(self, order_no, amount, product_name, payment_type='alipay', 
                      notify_url=None, return_url=None):
        """
        创建支付订单
        
        Args:
            order_no: 商户订单号
            amount: 支付金额（元）
            product_name: 商品名称
            payment_type: 支付方式（alipay/wxpay）
            notify_url: 异步通知地址
            return_url: 同步返回地址
        
        Returns:
            {
                'success': True/False,
                'pay_url': '支付链接',
                'qr_code': '二维码链接',
                'error': '错误信息'
            }
        """
        # 构建参数
        params = {
            'pid': self.merchant_id,
            'type': payment_type,  # alipay 或 wxpay
            'out_trade_no': order_no,
            'name': product_name,
            'money': f"{amount:.2f}",
            'notify_url': notify_url or f"https://yourdomain.com/api/payment/notify",
            'return_url': return_url or f"https://yourdomain.com/payment/success",
            'sign_type': 'MD5'
        }
        
        # 生成签名
        params['sign'] = self.generate_sign(params)
        
        try:
            # 调用API接口
            response = requests.post(self.api_url, data=params, timeout=10)
            result = response.json()
            
            if result.get('code') == 1:
                # 成功
                return {
                    'success': True,
                    'pay_url': result.get('payurl', ''),
                    'qr_code': result.get('qrcode', ''),
                    'order_no': order_no
                }
            else:
                return {
                    'success': False,
                    'error': result.get('msg', '创建支付订单失败')
                }
        except Exception as e:
            return {
                'success': False,
                'error': f'支付接口调用异常: {str(e)}'
            }
    
    def verify_notify(self, params):
        """验证异步通知签名"""
        # 获取通知中的签名
        notify_sign = params.get('sign', '')
        
        # 重新计算签名
        calculated_sign = self.generate_sign(params)
        
        # 验证签名
        return notify_sign.lower() == calculated_sign.lower()
```

## 🔐 五、安全注意事项

### 1. **签名验证**

**必须验证签名**，防止伪造支付通知：

```python
def verify_notify(params):
    """验证通知签名"""
    notify_sign = params.get('sign', '')
    calculated_sign = generate_sign(params)
    return notify_sign.lower() == calculated_sign.lower()
```

### 2. **订单号唯一性**

- 订单号必须全局唯一
- 建议格式：`ORDER + 时间戳 + 随机字符串`
- 避免重复订单号导致支付混乱

### 3. **金额验证**

- 验证支付金额与订单金额一致
- 防止金额篡改

### 4. **幂等性处理**

- 异步通知可能重复发送
- 需要检查订单是否已处理
- 已处理的订单直接返回 `success`

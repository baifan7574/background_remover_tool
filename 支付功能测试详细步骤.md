# 💳 支付功能测试详细步骤

## 📋 测试前准备

### 1. 确认环境

- [ ] 后端服务器已启动
- [ ] 前端可以正常访问
- [ ] 已登录账号
- [ ] 码支付客户端已初始化（查看后端日志）

---

## 🎯 支付测试完整流程

### 步骤1：访问支付页面

**方式1：从首页进入**
1. 打开网站首页
2. 点击"升级会员"按钮
3. 或点击导航栏的"会员中心"

**方式2：直接访问**
```
http://localhost:8000/payment.html
```

---

### 步骤2：选择会员计划

1. **查看会员计划列表**
   - 免费版（¥0）
   - 入门版（¥19/月 或 ¥180/年）
   - 专业版（¥49/月 或 ¥450/年）
   - 旗舰版（¥99/月 或 ¥850/年）
   - 企业版（¥299/月 或 ¥2,500/年）

2. **选择要购买的会员计划**
   - 例如：点击"入门版"的"立即升级"按钮

---

### 步骤3：选择支付方式

1. **弹出支付模态框**
   - 应该看到会员详情
   - 价格信息
   - 支付方式选择

2. **选择月付/年付**
   - 点击"月付"或"年付"按钮
   - 查看价格变化

3. **选择支付方式**
   - 点击"支付宝"或"微信支付"图标
   - 选中的方式会高亮显示

---

### 步骤4：确认支付

1. **点击"确认支付"按钮**

2. **查看后端日志**（重要！）
   ```
   🔄 准备调用码支付接口...
   📤 调用码支付API: https://pay.mzfpay.com/xpay/epay/mapi.php
   📥 码支付响应状态码: 200
   ✅ 创建支付订单成功
   ```

3. **如果成功：**
   - 应该跳转到码支付页面
   - 或显示支付二维码

4. **如果失败：**
   - 查看后端日志中的错误信息
   - 告诉我具体的错误

---

### 步骤5：完成支付

1. **在码支付页面：**
   - 选择支付方式（支付宝/微信）
   - 完成支付

2. **支付成功后：**
   - 应该自动跳转回成功页面
   - 或手动访问：`http://localhost:8000/payment/success.html?order_no=订单号`

---

### 步骤6：验证支付结果

#### 方法1：查看后端日志

**应该看到：**
```
📨 收到支付回调通知: {...}
✅ 支付回调处理成功 - 订单号: ORDER...
✅ 支付成功并激活会员 - 用户: ..., 计划: ...
```

#### 方法2：查看订单状态

**访问调试接口：**
```
http://localhost:5000/api/payment/debug/list-orders
```

**查看订单状态：**
- `"status": "paid"` → ✅ 支付成功
- `"status": "pending"` → ⏳ 待支付

#### 方法3：查看会员状态

1. 登录网站
2. 点击右上角用户头像 → "个人资料"
3. 查看会员信息：
   - 会员等级应该已升级
   - 会员到期时间应该有未来日期

## ⚠️ 常见问题

### 问题1：点击"确认支付"没有反应

**检查：**
1. 打开浏览器开发者工具（F12）
2. 查看Console标签，是否有错误
3. 查看Network标签，检查API请求是否发送

**可能原因：**
- JavaScript错误
- API请求失败
- Token无效

---

### 问题2：创建订单返回500错误

**检查后端日志：**

**如果看到：**
```
❌ 调用码支付接口异常: ...
```

**可能原因：**
- 网络连接问题
- 码支付API地址错误
- 签名错误
- 参数错误

**解决方法：**
- 查看完整的错误信息
- 告诉我具体的错误，我帮您诊断

---

### 问题3：支付成功后没有回调

**这是正常的（本地环境）：**

**原因：**
- 本地地址（localhost）无法接收码支付的回调
- 码支付无法访问您的本地服务器

**解决方法：**
1. **使用ngrok（测试用）：**
   ```bash
   ngrok http 5000
   ```
   然后在码支付平台配置回调地址为ngrok提供的地址

2. **手动验证支付：**
   - 支付完成后，手动访问成功页面
   - 或手动调用激活会员接口

3. **部署到服务器后：**
   - 配置真实的回调地址
   - 就可以自动接收回调了

---

### 问题4：支付成功但会员未激活

**检查：**
1. 后端日志中是否有回调通知
2. 回调处理是否成功
3. 数据库中的订单状态

**如果本地环境无法接收回调：**
- 这是正常的
- 部署到服务器后配置回调地址即可

## 🔍 问题分析

### 可能的原因：

1. **return_url 包含查询参数**
   - 当前包含：`?out_trade_no=...&order_no=...`
   - 可能导致签名计算错误

2. **签名算法可能不同**
   - 可能要求大写MD5
   - 可能key格式不同

3. **参数值格式问题**
   - 商品名称包含中文
   - URL参数格式问题

---

## ✅ 修复方案

### 修复1：简化 return_url ✅

已修改：移除了重复的查询参数，只保留 `order_no`

### 修复2：尝试不同的签名格式

如果简化后仍有问题，需要尝试：

**方式A：大写MD5**
```python
sign = hashlib.md5(sign_str.encode('utf-8')).hexdigest().upper()
```

**方式B：不同的key格式**
```python
# 尝试1：&key=商户密钥（当前方式）
```

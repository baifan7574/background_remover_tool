# 🔧 解决 localhost:8000 无法访问的问题

## ✅ 我检查到的状态

- **后端服务器（5000端口）**：✅ 正在运行
- **前端服务器（8000端口）**：❌ 没有运行（这就是无法访问的原因）

---

## 🎯 解决方法

### 方法1：使用启动脚本（最简单）

1. **确保之前的服务器窗口都关闭了**
2. **双击 `start_dev_servers.bat` 文件**
3. **等待两个窗口都打开**
4. **检查前端窗口**：应该显示类似这样的信息：
   ```
   Serving HTTP on :: port 8000 ...
   ```
5. **在浏览器打开**：`http://localhost:8000`

---

### 方法2：手动启动前端服务器

**如果启动脚本有问题，手动启动：**

1. **打开一个命令行窗口**（Windows键+R，输入cmd）
2. **输入以下命令**：
   ```bash
   cd D:\background_remover_tool\frontend
   python -m http.server 8000
   ```
3. **等待显示**：
   ```
   Serving HTTP on :: port 8000 ...
   ```
4. **保持这个窗口打开**（不要关闭）
5. **在浏览器打开**：`http://localhost:8000`

---

## 🔍 检查步骤

### 1. 检查是否有两个窗口在运行

**应该有：**
- ✅ 窗口1：显示 "🚀 启动独立测试版应用..." （后端）
- ✅ 窗口2：显示 "Serving HTTP on :: port 8000 ..." （前端）

**如果只有一个窗口：**
- 说明前端服务器没有启动成功
- 使用上面的方法2手动启动

---

### 2. 检查端口是否被占用

**打开新的命令行窗口，输入：**
```bash
netstat -ano | findstr :8000
```

**如果显示有内容：** 说明端口8000正在使用（好的）
**如果没有显示：** 说明前端服务器没有启动

---

- **如果窗口闪退**：可能是路径问题或Python问题

## ❓ 常见问题

### 问题1：窗口一闪就关闭了

**原因**：可能是路径错误或Python问题

**解决**：
1. 手动打开命令行
2. 先 `cd D:\background_remover_tool\frontend`
3. 然后 `python -m http.server 8000`
4. 看看是否有错误信息

---

### 问题2：显示"端口已被占用"

**原因**：8000端口被其他程序占用

**解决**：
1. 关闭占用8000端口的程序
2. 或者换个端口：
   ```bash
   python -m http.server 8001
   ```
3. 然后访问 `http://localhost:8001`

---

### 问题3：显示"找不到模块"

**原因**：Python环境问题

**解决**：
```bash
python -m pip install --upgrade pip
python --version  # 确认Python版本
```

### 测试3：模拟支付成功

#### 方式1：手动调用回调接口（推荐用于测试）

**步骤3.1：获取订单号**

从测试1中记录的订单号，或从订单列表获取。

**步骤3.2：构造回调请求**

**使用curl：**
```bash
curl -X POST http://localhost:5000/api/payment/notify \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "out_trade_no=订单号&trade_status=TRADE_SUCCESS&total_amount=19.00&sign=测试签名"
```

**使用Postman：**
1. 方法：`POST`
2. URL：`http://localhost:5000/api/payment/notify`
3. Headers：`Content-Type: application/x-www-form-urlencoded`
4. Body（form-data）：
   - `out_trade_no`: `你的订单号`
   - `trade_status`: `TRADE_SUCCESS`
   - `total_amount`: `19.00`
   - `sign`: `测试签名`（可以随便填，测试环境）

**步骤3.3：检查回调结果**

**预期结果：**
- ✅ 后端日志显示：`支付回调处理成功`
- ✅ 订单状态更新为 `paid`
- ✅ 用户会员等级更新

**检查后端日志：**
```
✅ 支付回调处理成功
   订单号: ORDER_2024xxxxxx
   状态: paid
   会员激活: basic
```

## 🔍 常见问题排查

### 问题1：订单创建失败（500错误）

**检查：**
1. 后端日志中的错误信息
2. 码支付配置是否正确
3. 网络连接是否正常

**可能原因：**
- 码支付API调用失败
- 网络超时
- 配置错误

**解决方法：**
- 检查 `backend/mzfpay_client.py` 中的错误处理
- 查看后端日志的详细错误信息

---

### 问题2：支付页面不跳转

**检查：**
1. 前端是否正确处理支付响应
2. 码支付返回的数据格式
3. 浏览器控制台错误

**可能原因：**
- 码支付返回格式不符合预期
- 前端JavaScript错误
- CORS问题

---

### 问题3：支付回调不生效

**检查：**
1. 回调接口是否被调用
2. 回调数据格式是否正确
3. 签名验证是否通过

**可能原因：**
- 回调URL配置错误（生产环境）
- 签名验证失败
- 回调数据格式不正确

---

### 问题4：会员未激活

**检查：**
1. 订单状态是否为 `paid`
2. 会员激活API是否被调用
3. 用户资料是否更新

**可能原因：**
- 支付回调未正确处理
- 会员激活逻辑错误
- 数据保存失败

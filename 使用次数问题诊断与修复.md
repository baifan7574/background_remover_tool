# 🔍 使用次数问题诊断与修复

## ❌ 问题描述

**用户反馈：**
- 注册新用户后，使用背景移除工具
- 应该扣除1次使用次数
- 但实际没有扣除，显示次数没有变化

**怀疑原因：**
- 可能有两套使用次数系统
- 前后端不一致
- 检查逻辑和记录逻辑不匹配

---

## 🔍 问题分析

### 当前逻辑流程

1. **检查使用次数** (`check_daily_usage_limit`)
   - 从 `profile['daily_usage'][today][tool_name]` 获取今日使用次数
   - 检查是否超过限制

2. **记录使用次数** (`record_daily_usage`)
   - 优先使用邀请奖励
   - 如果没有奖励，调用 `data_manager.record_usage()`
   - `data_manager.record_usage()` 更新 `profile['daily_usage'][today][tool_name]`

### 可能的问题

#### 问题1：日期格式不一致

**检查：**
- `check_daily_usage_limit` 使用：`get_today_date()` → `'YYYY-MM-DD'`
- `data_manager.record_usage` 使用：`datetime.now().strftime('%Y-%m-%d')` → `'YYYY-MM-DD'`
- ✅ 格式一致

#### 问题2：数据结构不一致

**检查：**
- `check_daily_usage_limit` 读取：`daily_usage.get(today, {}).get(tool_name, 0)`
- `data_manager.record_usage` 写入：`profile['daily_usage'][today][tool_name] = current_count + 1`
- ✅ 结构一致

#### 问题3：数据未保存

**检查：**
- `data_manager.record_usage()` 调用 `self.save_all()`
- ✅ 应该已保存

#### 问题4：邀请奖励逻辑问题

**可能原因：**
- `record_daily_usage` 优先使用邀请奖励
- 如果使用了邀请奖励，可能没有更新 `daily_usage` 计数
- 导致 `check_daily_usage_limit` 读取的计数不准确

---

## 🐛 发现的问题

### 问题：邀请奖励使用后，daily_usage 计数未更新

**代码位置：** `sk_app.py` 第399-439行

**问题描述：**
```python
def record_daily_usage(user_id, tool_name):
    # 如果使用邀请奖励
    if 使用一次性奖励:
        reward['used'] = used + 1
        return True, "使用邀请奖励"  # ❌ 直接返回，没有更新 daily_usage
    
    # 只有没有奖励时才更新 daily_usage
    success, message = data_manager.record_usage(user_id, tool_name)
```

**影响：**
- 使用邀请奖励时，`daily_usage` 计数不增加
- 但 `check_daily_usage_limit` 读取的是 `daily_usage` 计数
- 导致显示的使用次数不准确

---

## ✅ 修复方案

### 方案1：使用邀请奖励时也更新 daily_usage（推荐）

**逻辑：**
- 使用邀请奖励时，仍然更新 `daily_usage` 计数
- 这样 `check_daily_usage_limit` 读取的计数是准确的
- 邀请奖励只是增加可用次数，不影响计数显示

### 方案2：check_daily_usage_limit 也考虑邀请奖励已使用次数

**逻辑：**
- `check_daily_usage_limit` 读取时，也检查邀请奖励的已使用次数
- 将邀请奖励已使用次数也计入总使用次数

---

## 🔧 实施修复

我将采用**方案1**，因为：
1. 更简单直接
2. 计数逻辑统一
3. 前端显示更准确

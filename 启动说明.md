# 🚀 系统启动说明

## 📖 前端和后端的关系

### 前端（Frontend）
- **位置**: `frontend/` 文件夹
- **作用**: 用户看到的网页界面（就像网站的"门面"）
- **包含**: HTML页面、CSS样式、JavaScript交互代码
- **特点**: 运行在用户的浏览器中

### 后端（Backend）  
- **位置**: `sk_app.py` 文件
- **作用**: 处理业务逻辑、连接数据库、提供API接口
- **特点**: 运行在服务器上，前端通过API调用后端功能

## 🔄 用户访问流程

```
用户打开网页 (前端)
    ↓
点击"背景移除"按钮
    ↓
前端发送请求到后端API (http://localhost:5000/api/tools/remove-background)
    ↓
后端处理图片（使用rembg库移除背景）
    ↓
后端返回处理结果给前端
    ↓
前端显示处理后的图片给用户
```

## ⚡ 快速启动（最简单的方式）

### 方式1：一键启动（推荐）

**Windows系统：**
1. 双击 `start_dev_servers.bat` 文件
2. 等待两个窗口启动完成
3. 在浏览器打开：`http://localhost:8000`

**Mac/Linux系统：**
```bash
bash start_dev_servers.sh
```

### 方式2：手动启动

#### 步骤1：启动后端服务器
打开一个命令行窗口：
```bash
cd D:\background_remover_tool
python sk_app.py
```
看到以下信息说明启动成功：
```
🚀 启动独立测试版应用...
✅ 数据持久化管理器已加载
🌐 访问地址: http://localhost:5000
```

#### 步骤2：启动前端服务器（新开一个命令行窗口）
```bash
cd D:\background_remover_tool\frontend
python -m http.server 8000
```
看到以下信息说明启动成功：
```
Serving HTTP on :: port 8000 ...
```

#### 步骤3：打开浏览器访问
在浏览器中打开：`http://localhost:8000`

## 📋 检查清单

### ✅ 启动前检查
- [ ] Python已安装（运行 `python --version` 检查）
- [ ] 依赖已安装（运行 `pip install -r requirements.txt`）
- [ ] 端口5000和8000未被占用

### ✅ 启动后检查
- [ ] 后端：访问 `http://localhost:5000/health` 应该看到JSON响应
- [ ] 前端：访问 `http://localhost:8000` 应该看到网页
- [ ] 打开浏览器控制台（F12），检查是否有错误

## 🔧 常见问题

### 问题1：端口被占用
**解决**: 修改端口号
- 后端：在 `sk_app.py` 最后一行修改 `port=5000` 为其他端口（如5001）
- 前端：修改启动命令为 `python -m http.server 8001`

### 问题2：前端无法连接后端
**解决**: 检查 `frontend/js/auth.js` 和 `frontend/js/app.js` 中的 `apiBaseUrl` 是否与后端端口一致

### 问题3：数据库连接错误
**解决**: 
- 如果使用Supabase，创建 `.env` 文件配置数据库
- 如果不想用Supabase，系统会自动使用本地JSON存储（数据保存在 `data/` 文件夹）

## 🎯 总结

**你需要启动两个服务：**
1. **后端API服务器** (sk_app.py) - 处理数据和处理请求
2. **前端网页服务器** (http.server) - 提供网页界面

**用户只需要访问：**
- `http://localhost:8000` (前端网页)

前端会自动调用后端的API来处理功能！

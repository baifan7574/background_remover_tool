# 📖 简单操作指南 - 解决支付签名错误

## 🎯 您现在要做什么？

**目标：修复支付签名错误，让支付功能正常工作**

---

## 📝 第一步：重启后端服务器

### 为什么要重启？
- 因为我们修改了代码
- 需要重新加载新的代码

### 怎么重启？

1. **找到后端服务器窗口**
   - 应该有一个命令行窗口标题是"后端服务器-端口5000"
   - 或者运行 `python sk_app.py` 的窗口

2. **停止服务器**
   - 按 `Ctrl + C` 停止当前服务器

3. **重新启动**
   ```bash
   python sk_app.py
   ```
   
   或者双击：`启动后端服务器.bat`

4. **等待启动完成**
   - 看到 "Running on http://127.0.0.1:5000" 就成功了

---

## 📝 第二步：测试支付功能

### 操作步骤：

1. **打开浏览器**
   - 访问：`http://localhost:8000`

2. **登录账号**
   - 如果没有账号，先注册一个

3. **选择会员计划**
   - 点击"升级会员"或访问支付页面
   - 选择一个会员计划（比如"入门版"）
   - 点击"立即升级"

4. **创建支付订单**
   - 选择"月付"或"年付"
   - 选择"支付宝"或"微信支付"
   - 点击"确认支付"

5. **查看后端日志**
   - 打开后端服务器的命令行窗口
   - 查看输出的信息

---

## 🔍 第三步：查看签名信息

### 应该看到什么？

**如果修复成功，应该看到：**
```
🔐 签名原始字符串: money=299.00&name=企业版-月付&...
🔐 MD5(小写): xxxxxx
🔐 MD5(大写): XXXXXX
📊 码支付接口返回结果: {'success': True, ...}
```

**如果还有签名错误，应该看到：**
```
❌ 码支付返回错误: 签名错误
```

---

## 🛠️ 第四步：如果还有签名错误

### 请告诉我：

1. **签名字符串是什么？**
   - 从后端日志中复制完整的签名字符串
   - 类似这样：`money=299.00&name=企业版-月付&...`

2. **码支付返回什么错误？**
   - 是"签名错误"还是其他错误？

3. **参数列表**
   - 从日志中看到的所有参数

---

## 📋 快速检查清单

在测试之前，确认：

- [ ] 后端服务器已重启（使用最新代码）
- [ ] 前端服务器正在运行（端口8000）
- [ ] 浏览器已打开并访问了 `http://localhost:8000`
- [ ] 已登录账号
- [ ] 后端日志窗口可以看到输出信息

---

## 🚀 最简单的方式（一键启动）

如果您想一次性启动所有东西：

1. **双击运行：** `start_dev_servers.bat`
   - 这会自动启动后端和前端服务器

2. **等待两个窗口打开**
   - "后端服务器-端口5000"
   - "前端服务器-端口8000"

3. **打开浏览器**
   - 访问：`http://localhost:8000`

4. **开始测试**

# 🔧 支付签名问题调试与修复

## ❌ 当前问题

**错误信息：** `签名错误`

**从日志看到的签名字符串：**
```
money=299.00&name=企业版-月付&notify_url=http://localhost:5000/api/payment/notify&out_trade_no=ORDER20251124184624E91AF07C&pid=10294&return_url=http://localhost:8000/payment/success.html?out_trade_no=ORDER20251124184624E91AF07C&order_no=ORDER20251124184624E91AF07C&type=wxpay&key=X0cJyf2G0EjDKtQe9NMf
```

---

## 🔍 问题分析

### 发现的问题：

1. **return_url 中包含多个查询参数**
   - 当前：`?out_trade_no=...&order_no=...`
   - 这可能导致签名计算错误

2. **可能的解决方案：**
   - **方案1**：简化 return_url，只保留 order_no
   - **方案2**：对URL参数进行URL编码
   - **方案3**：移除 return_url（如果不需要同步跳转）

---

## ✅ 已实施的修复

### 修复1：简化 return_url

**修改位置：** `sk_app.py` 第3362行

**修改前：**
```python
return_url = f"{frontend_base}/payment/success.html?out_trade_no={order_no}&order_no={order_no}"
```

**修改后：**
```python
return_url = f"{frontend_base}/payment/success.html?order_no={order_no}"
```

**原因：**
- 减少查询参数，避免影响签名
- `order_no` 已经包含所有必要信息

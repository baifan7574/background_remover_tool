# ✅ 支付成功验证指南

## 🔍 验证方法（从易到难）

### 方法1：检查后端日志（最简单）

**步骤：**
1. 查看后端服务器终端窗口（运行 `python sk_app.py` 的窗口）
2. 支付完成后，查找以下日志：

**支付成功的标志：**
```
📨 收到支付回调通知: {...}
✅ 支付回调处理成功 - 订单号: ORDER...
✅ 支付成功并激活会员 - 订单号: ..., 用户: ..., 计划: ...
```

**如果没有看到回调通知：**
```
⚠️ 可能是：
- 支付还未完成
- 回调地址无法访问（本地环境）
- 支付失败
```

---

### 方法2：检查会员状态（最直观）

**步骤：**
1. 登录网站
2. 点击右上角用户头像 → "个人资料"
3. 查看会员信息：
   - **会员等级**：应该显示您购买的会员等级（如：入门版、专业版）
   - **会员到期时间**：应该显示未来的日期
   - **使用统计**：应该显示新的使用限制

**如果会员已激活：**
```
✅ 会员等级：入门版（或您购买的等级）
✅ 会员到期时间：2024-12-17（未来的日期）
✅ 使用限制已增加
```

**如果会员未激活：**
```
❌ 会员等级：免费版
❌ 会员到期时间：无
❌ 使用限制未增加
```

### 方法3：手动查询订单状态（最准确）

#### 方法3.1：通过浏览器访问API

**步骤：**
1. 打开浏览器，访问以下URL（替换为您的订单号）：
   ```
   http://localhost:5000/api/payment/query-order/ORDER20241117123456789
   ```
   
2. 需要先获取Token：
   - 打开浏览器开发者工具（F12）
   - 打开 Console 标签
   - 输入：`localStorage.getItem('token')`
   - 复制显示的token值

3. 在另一个浏览器标签页，使用以下方法：

**使用 curl（如果有）：**
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:5000/api/payment/query-order/ORDER20241117123456789
```

**使用浏览器扩展（如 Postman 或 REST Client）：**
- URL: `http://localhost:5000/api/payment/query-order/ORDER20241117123456789`
- Method: `GET`
- Headers: `Authorization: Bearer YOUR_TOKEN`

**预期返回：**
```json
{
  "success": true,
  "order": {
    "order_no": "ORDER20241117123456789",
    "status": "paid",  // 如果支付成功，应该是 "paid"
    "amount": 19,
    "plan": "basic",
    "paid_at": "2024-11-17T12:34:56",
    "trade_no": "码支付的订单号"
  }
}
```

**状态说明：**
- `status: "pending"` → 支付中或未支付
- `status: "paid"` → ✅ 支付成功
- `status: "cancelled"` → 订单已取消
- `status: "expired"` → 订单已过期

---

#### 方法3.2：在浏览器控制台查询

**步骤：**
1. 打开网站首页（`http://localhost:8000`）
2. 按 `F12` 打开开发者工具
3. 切换到 **Console** 标签
4. 输入以下代码（替换订单号）：

```javascript
// 替换为您的订单号
const orderNo = 'ORDER20241117123456789';
const token = localStorage.getItem('token');

fetch(`http://localhost:5000/api/payment/query-order/${orderNo}`, {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
})
.then(res => res.json())
.then(data => {
  console.log('订单状态:', data);
  if (data.success && data.order.status === 'paid') {
    console.log('✅ 支付成功！');
    console.log('订单号:', data.order.order_no);
    console.log('金额:', data.order.amount);
    console.log('会员计划:', data.order.plan);
    console.log('支付时间:', data.order.paid_at);
  } else {
    console.log('❌ 支付未成功或订单不存在');
  }
})
.catch(error => console.error('查询失败:', error));
```

5. 按 `Enter` 执行
6. 查看输出结果

### 方法4：检查码支付平台订单（最可靠）

**步骤：**
1. 登录码支付平台：`pay.mzfpay.com`
2. 进入 **"订单管理"** 页面
3. 查找您的订单号（`ORDER...`）
4. 查看订单状态：
   - **已支付** → ✅ 支付成功
   - **待支付** → ⏳ 还未支付
   - **已关闭** → ❌ 支付失败或超时

**注意事项：**
- 码支付平台的订单号可能和您的不一样（有 `trade_no` 字段）
- 可以搜索商户订单号（`out_trade_no`）来查找

### 方法5：检查后端内存中的订单数据

**这个方法需要修改代码，临时添加一个调试接口**

如果您熟悉代码，可以在 `sk_app.py` 中临时添加：

```python
@app.route('/api/payment/debug-orders', methods=['GET'])
def debug_orders():
    """调试接口：查看所有订单（仅用于开发测试）"""
    return jsonify({
        'orders': payment_orders,
        'count': len(payment_orders)
    })
```

然后访问：`http://localhost:5000/api/payment/debug-orders`

**⚠️ 警告：** 这个接口会暴露所有订单信息，仅用于开发测试，不要在生产环境使用！

## 🎯 快速验证脚本

创建一个简单的HTML文件来验证：

**文件名：`check_payment.html`**

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>支付验证工具</title>
</head>
<body>
    <h1>支付验证工具</h1>
    <input type="text" id="orderNo" placeholder="输入订单号" style="width:300px; padding:10px;">
    <button onclick="checkOrder()" style="padding:10px;">查询订单</button>
    <div id="result" style="margin-top:20px;"></div>

    <script>
        async function checkOrder() {
            const orderNo = document.getElementById('orderNo').value;
            if (!orderNo) {
                alert('请输入订单号');
                return;
            }

            const token = localStorage.getItem('token');
            if (!token) {
                alert('请先登录网站，获取token');
                return;
            }

            try {
                const response = await fetch(`http://localhost:5000/api/payment/query-order/${orderNo}`, {
                    headers: {
                        'Authorization': `Bearer ${token}`,
                        'Content-Type': 'application/json'
                    }
                });

                const data = await response.json();
                const resultDiv = document.getElementById('result');

                if (data.success && data.order) {
                    const order = data.order;
                    let html = '<h3>订单信息</h3>';
                    html += `<p><strong>订单号：</strong>${order.order_no}</p>`;
                    html += `<p><strong>状态：</strong>${order.status === 'paid' ? '✅ 已支付' : '⏳ ' + order.status}</p>`;
                    html += `<p><strong>金额：</strong>¥${order.amount}</p>`;
                    html += `<p><strong>会员计划：</strong>${order.plan}</p>`;
                    if (order.paid_at) {
                        html += `<p><strong>支付时间：</strong>${order.paid_at}</p>`;
                    }
                    resultDiv.innerHTML = html;
                } else {
                    resultDiv.innerHTML = `<p style="color:red;">查询失败：${data.error || '未知错误'}</p>`;
                }
            } catch (error) {
                document.getElementById('result').innerHTML = `<p style="color:red;">错误：${error.message}</p>`;
            }
        }
    </script>
</body>
</html>
```

**使用方法：**
1. 将这个文件保存到 `frontend` 目录
2. 访问：`http://localhost:8000/check_payment.html`
3. 输入订单号，点击"查询订单"

## 🔧 常见问题

### Q1: 后端日志中没有回调通知，但码支付显示已支付？

**A:** 可能是本地环境无法接收回调。解决：
- 使用 ngrok 等内网穿透工具
- 或手动激活会员（通过API）

### Q2: 订单状态显示 `pending`，但已支付？

**A:** 可能是回调未处理。解决：
- 检查后端日志
- 检查回调接口是否正常
- 检查签名验证是否通过

### Q3: 如何获取订单号？

**A:** 
- 查看后端日志中创建订单时的输出
- 查看浏览器 Network 请求中的响应
- 查看码支付平台的订单列表

## 二、服务器部署

### 2.1 连接服务器

**使用SSH连接：**
```bash

```

**或使用密码登录：**

### 2.2 安装基础环境

#### 步骤1：更新系统
```bash
# Ubuntu/Debian
sudo apt update
sudo apt upgrade -y

# CentOS/RHEL
sudo yum update -y
```

#### 步骤2：安装Python 3.8+
```bash
# Ubuntu/Debian
sudo apt install python3 python3-pip python3-venv -y

# CentOS/RHEL
sudo yum install python3 python3-pip -y

# 验证安装
python3 --version
pip3 --version
```

#### 步骤3：安装Nginx（Web服务器）
```bash
# Ubuntu/Debian
sudo apt install nginx -y

# CentOS/RHEL
sudo yum install nginx -y

# 启动Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

#### 步骤4：安装MySQL（如果选择MySQL）
```bash
# Ubuntu/Debian
sudo apt install mysql-server -y

# CentOS/RHEL
sudo yum install mysql-server -y

# 启动MySQL
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation
```

## 三、数据库设置

### 3.1 创建数据库（MySQL示例）

**登录MySQL：**
```bash
sudo mysql -u root -p
```

**创建数据库和用户：**
```sql
-- 创建数据库
CREATE DATABASE background_remover_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'your_secure_password';

-- 授权
GRANT ALL PRIVILEGES ON background_remover_db.* TO 'app_user'@'localhost';
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

---

### 3.2 创建数据库表

**创建初始化SQL脚本：**

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
    id VARCHAR(50) PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(255),
    plan VARCHAR(50) DEFAULT 'free',
    membership_expires_at DATETIME,
    invite_code VARCHAR(50) UNIQUE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 订单表
CREATE TABLE IF NOT EXISTS payment_orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_no VARCHAR(100) UNIQUE NOT NULL,
    user_id VARCHAR(50) NOT NULL,
    plan VARCHAR(50) NOT NULL,
    plan_name VARCHAR(255),
    billing_period VARCHAR(20),
    duration_days INT,
    amount DECIMAL(10, 2) NOT NULL,
    payment_method VARCHAR(50),
    status VARCHAR(50) DEFAULT 'pending',
    trade_no VARCHAR(100),
    paid_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    expires_at DATETIME,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 邀请记录表
CREATE TABLE IF NOT EXISTS invite_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    invite_code VARCHAR(50) NOT NULL,
    inviter_id VARCHAR(50) NOT NULL,
    invitee_id VARCHAR(50),
    status VARCHAR(50) DEFAULT 'pending',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (inviter_id) REFERENCES users(id),
    FOREIGN KEY (invitee_id) REFERENCES users(id)
);

-- 使用记录表
CREATE TABLE IF NOT EXISTS usage_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL,
    tool_name VARCHAR(100) NOT NULL,
    usage_date DATE NOT NULL,
    usage_count INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    UNIQUE KEY unique_user_tool_date (user_id, tool_name, usage_date)
);

-- 创建索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_invite_code ON users(invite_code);
CREATE INDEX idx_orders_user_id ON payment_orders(user_id);
CREATE INDEX idx_orders_order_no ON payment_orders(order_no);
CREATE INDEX idx_orders_status ON payment_orders(status);
CREATE INDEX idx_usage_user_date ON usage_records(user_id, usage_date);
```

**执行SQL脚本：**
```bash
mysql -u app_user -p background_remover_db < init_database.sql
```

## 六、常见问题解决

### 6.1 数据库连接失败

**问题：** `数据库连接失败`

**解决：**
1. 检查数据库是否启动：`sudo systemctl status mysql`
2. 检查用户名密码是否正确
3. 检查数据库是否存在
4. 检查防火墙是否允许连接

---

### 6.2 支付回调无法接收

**问题：** 支付成功后没有回调

**解决：**
1. 检查回调地址是否正确配置
2. 检查服务器防火墙是否开放端口
3. 检查Nginx配置是否正确
4. 查看后端日志确认是否收到请求

---

### 6.3 静态文件无法访问

**问题：** CSS/JS文件404

**解决：**
1. 检查Nginx配置
2. 检查文件路径是否正确
3. 检查文件权限

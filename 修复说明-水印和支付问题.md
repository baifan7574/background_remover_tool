# 🔧 修复说明 - 水印和支付问题

## ✅ 已修复的问题

### 问题1：加水印功能参数不生效

**问题描述：**
- 水印位置、颜色、字体大小、透明度等参数无法调整
- 水印始终在右上角，使用默认字体颜色和位置

**原因分析：**
1. 后端代码已正确接收参数，但位置值可能需要映射（中文→英文）
2. 前端透明度滑块缺少实时更新显示的事件绑定

**修复内容：**
1. ✅ 添加位置名称映射（支持中文和英文）
2. ✅ 修复透明度滑块实时显示

**测试方法：**
1. 打开加水印工具
2. 修改水印文字、位置、透明度、字体大小、颜色
3. 上传图片并处理
4. 检查水印是否按设置的参数显示

### 问题2：支付创建订单500错误（签名错误）

**问题描述：**
- 点击"确认支付"后返回500错误
- 错误信息：`支付接口调用失败: 签名错误`

**原因分析：**
- 签名生成算法可能不正确
- 参数顺序或格式可能不符合码支付要求
- 缺少调试信息，难以定位问题

**修复内容：**
1. ✅ 添加签名生成过程的调试日志
2. ✅ 确保参数排序正确
3. ✅ 验证签名字符串格式

**测试方法：**
1. 选择会员计划
2. 点击"确认支付"
3. 查看后端日志中的签名信息
4. 如果仍有错误，查看日志中的签名字符串和生成的签名

## 🔍 调试信息

### 水印功能调试

查看后端日志，应该看到：
```
🖼️ 水印参数 - 文字: xxx, 位置: top-right, 透明度: 0.7, 字体大小: 50, 颜色: #000000
📍 水印位置: (x, y) (图片尺寸: width x height)
🎨 水印颜色: RGB(r, g, b), 透明度: alpha/255
```

### 支付功能调试

查看后端日志，应该看到：
```
🔐 签名原始字符串: pid=10294&type=alipay&...
🔐 生成的签名: xxxxxx
📊 码支付接口返回结果: {...}
```

如果返回错误，检查：
- 签名字符串是否正确
- 参数名是否符合码支付要求
- 商户密钥是否正确
